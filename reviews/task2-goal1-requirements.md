# Task 2 Goal 1 requirements traceability matrix

> **Status: Current review.** Assignment-derived traceability aid. The Team 6 assignment remains the authoritative requirement source; this matrix is not a requirement source or accepted decision record.

## Scope and classification

This matrix is derived only from the Team 6 SoftwareLab assignment PDF. It records explicit Task 2 Goal 1 requirements and separates post-gate work so it is not mistaken for Goal 1 acceptance.

- **Mandatory**: the assignment directs the team to do or provide it.
- **Conditional**: required only under the condition stated in the assignment, or expressed there as a recommended default rather than an unconditional rule.
- **Nice-to-have**: explicitly optional, “nice-to-have,” or “ideal.”
- A dash means the assignment states no separate item for that field.

The proposed mapping is verified: **1A = component 1 (reader/writer), 1B = component 2 (LichtFeld workflow), 1C = component 3 (measurement harness), 1D = component 4 (research gate), and the selected refinement prototype begins only after that gate as Goal 2 component 5.**
