# .dependency-cruiser.cjs

## Purpose

Dependency-cruiser rules for the landing sources (`check:boundaries` in the
`check:all` gate), mirroring the sibling apps: no unresolvable imports
(typo catcher) and no circular dependencies within `src/`.

## Invariants & assumptions

- Scoped to `src/` (`includeOnly`), does not follow `node_modules`.
- The intended layering is flat: `main.ts` → feature modules →
  `scene/*` → three.js/anime.js. Cycles would most plausibly appear
  between `scene/story-timeline.ts` and `scene/scene-controller.ts` —
  keep the timeline module free of controller imports.

## Usage

```bash
pnpm run check:boundaries   # depcruise -c config/.dependency-cruiser.cjs src
```

## Tests

No unit test; exercised by the `check:all` gate inside `pnpm test`.
