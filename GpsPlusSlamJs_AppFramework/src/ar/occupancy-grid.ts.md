# AR-Space Occupancy Grid

## Purpose

TS port of the Unity voxel grid (`PointCloudData.cs` in GpsPlusSlamUnity): folds the persisted depth-sample stream (`recording/recordDepthSample`) into a sparse 3D grid of occupied cells in **raw WebXR space**, with free-space carving along each camera→point ray. Plain in-memory class — no THREE/DOM/Redux. Derived state: fed by store subscribers; the action stream remains the persisted source of truth (same pattern as `wire-frame-tile-subscribers` in the RecorderApp).

Plan: `GpsPlusSlamJs_Docs/docs/2026-06-11-2134-depth-occupancy-grid-port-plan.md`.

## Public API

- **`new OccupancyGrid(options?)`** — `cellSizeM` (default 0.15, Unity parity), `carveStopCells` (default 2). Throws `RangeError` on non-positive/non-finite cell size or negative/non-integer carve stop.
- **`DEFAULT_OCCUPANCY_CELL_SIZE_M = 0.18`** / **`DEFAULT_OCCUPANCY_MIN_OBSERVATIONS = 3`** — the framework-level _recommended reconstruction defaults_ (voxel size + noise floor) that reconstruct-and-render apps (Recorder, PhysicsDemo) inherit, so both share one tuning source. **Deliberately separate from the constructor's `cellSizeM ?? 0.15` primitive fallback** — that stays 0.15 because many low-level tests build `new OccupancyGrid()` and assume 0.15 quantization; both apps always pass an explicit `cellSizeM`, so the two can differ. Tuning (2026-07-16 cellSize × noise corpus sweep): speed comes from the coarser 18 cm voxel (+18% early coverage vs the old 15 cm at equal fidelity); the noise floor stays at 3 because floaters (= phantom colliders) are governed by the floor, not the voxel size (mc 3 ≈ 1.9% vs mc 2 ≈ 3.5%). Supersedes the 2026-07-15 device-impression change that briefly shipped noise 2. `0.18` is within `OCCUPANCY_CONSTRAINTS.cellSizeM.max` (0.20); `3` within `minConfidence` 1–10.
- **`addSample(sample: DepthSample): number`** — unprojects each point (`unprojectDepthPoint`), carves free space, increments the point cell's observation count. Returns the number of points added. Points that cannot be unprojected (old recordings without `projectionMatrix`, invalid depth/coords) are skipped; non-finite camera positions skip the whole sample. Carving and incrementing run as **two passes** over the sample's points (all rays carved first, then all endpoints incremented) so the result is independent of point iteration order — see invariant 3.
- **`getOccupiedCells(minObservations = 1): GridCell[]`** — cells observed at least that often. Returns a **shared immutable snapshot**: the last `(revision, minObservations)` walk is memoized (Step 1.2 of the [2026-07-03 long-session fps plan](../../../../gps-plus-slam/GpsPlusSlamJs_Docs/docs/2026-07-03-1344-long-session-fps-and-voxel-grid-scaling-plan.md)), so the second of the two identical per-refresh snapshots (cubes + occluder, same minConfidence floor) is answered without a re-walk — callers must never mutate the returned array. Any occupied-set mutation (add/carve/clear) invalidates the memo via the revision; floors `> 10` bypass it (their threshold crossings don't bump the revision, see `getRevision`).
- **`getOccupiedCellsFlat(minObservations = 1): Int32Array`** — flat `[x0,y0,z0, x1,y1,z1, …]` variant for the mesh-worker pack path (Step 1.3 of the 2026-07-03 fps plan): `packMeshRequest` accepts it zero-copy and transfers its buffer to the worker, deleting the tuple intermediate. Same cells/order as the tuple API (property-tested), but a **fresh array every call** (the transfer detaches the buffer); when the tuple memo is warm it flattens from it without a Map re-walk.
- **`getOccupiedCellsWithin(centerPos, radiusM, minObservations = 1): GridCell[]`** / **`getOccupiedCellsWithinFlat(…): Int32Array`** — the viewer-local window (Step 2 of the 2026-07-03 fps plan): occupied cells whose **centers** lie within `radiusM` of `centerPos`, computed via the internal 16³ **chunk index** so the cost is O(cells within R + chunks touched), independent of total explored area. Result set ≡ the brute-force radius filter of `getOccupiedCells` (property-tested; iteration order may differ). Throws `RangeError` for a non-finite/non-positive radius; a non-finite center returns `[]` (tracking-glitch policy, mirrors `raycast`). Fresh arrays per call (no memo — the camera moves), flat variant transfer-safe like `getOccupiedCellsFlat`.
- **`getChunkRevision(chunk): number`** — per-chunk dirty revision (chunk coords = `floor(cell/16)` per axis; 0 = never touched). Bumped whenever a mutation inside the chunk could change some consumer's occupied set (same semantics as `getRevision`); survives the chunk emptying; reset by `clear`. This is the bookkeeping the [worker plan's Phase-2 dirty-chunk remesh sketch](../../../../gps-plus-slam/GpsPlusSlamJs_Docs/docs/2026-07-01-0733-occluder-worker-and-chunked-remesh-plan.md) needs — no remesh machinery lives here (explicit non-goal).
- **`cellForPosition(pos): GridCell`** — round-quantization per axis (−0 normalized).
- **`getCellCenter(cell): Vector3`** — `cell · cellSizeM` (the 15 cm-lattice center).
- **`getCellPoint(cell): Vector3 | null`** — running-average of the EXACT unprojected surface points observed in the cell (raw WebXR), or `null` for an unknown cell. Unlike `getCellCenter` this hugs the real measured surface and noise-averages across viewpoints; it always lies within `cellSizeM/2` of the cell center. Used by the COLMAP `points3D` export and the debug cubes (follow-up Item A); every observation contributes a position, so the divisor is the observation `count`.
- **`getCellColor(cell): RgbTuple | null`** — running-average camera color of the cell's COLORED observations (Iter 8), rounded and clamped to 0–255 per channel; `null` for unknown cells or cells only ever observed without `point.rgb` (visualizers fall back to height-based coloring). Color-less observations advance the observation count but never dilute the average; non-finite rgb triples are ignored defensively.
- **`raycast(startPos, endPos, minObservations = 1): Vector3 | null`** — center of the first sufficiently-observed cell on the Bresenham line (port of Unity `TryRaycast`; hook for future cursor/floor-detection parity). Returns `null` for non-finite input or no hit. Both `raycast` and the internal carving inherit `bresenham3d`'s `MAX_TRACE_STEPS` circuit breaker: a finite-but-absurd start/end (a tracking glitch, a corrupt `projectionMatrix` unprojecting to a huge world point) that spans more than ~150 km throws a `RangeError` instead of freezing the main thread.
- **`cellKey(cell): number`** / **`unpackCellKey(key): GridCell`** — the bijective numeric key packing and its exact inverse (property-tested over the full ±65 535 envelope). Exported since Step 3.1 of the 2026-07-03 fps plan: per-cell records no longer store their coordinate tuple, so every tuple a public API returns is recovered from the key.
- **`clear(): void`** / **`size: number`**.
- **`getRevision(): number`** — a monotonic version that changes only when the occupied set (at any `minConfidence ≤ 10`) could have changed: a cell added, a cell removed by carving, a cell's count rising while still `≤ 10` (a possible threshold crossing), or `clear` on a non-empty grid. Re-observing an already-**settled** cell (count `> 10`) does NOT bump it. A consumer (cube refresh / occluder re-mesh) caches the value it last rendered and **skips its O(cells) re-derive when the revision is unchanged** — the dominant idle-time saving over a long session (`wireOccupancyGridSubscribers` does this for both sinks).

## Invariants & Assumptions

1. **Raw WebXR frame everywhere** — `DepthSample.cameraPos`/points are raw WebXR (local-floor); no NUE conversion in this pipeline.
2. **Observation counts instead of Unity's render-buffer indices** — WebXR has no per-pixel confidence; `minObservations` is the noise filter.
3. **Carving** — Bresenham from camera cell to point cell, stopping `carveStopCells` dominant-axis steps before the endpoint (depth-noise tolerance). Deliberate deviations from Unity: carving is skipped when camera and point share a cell, and the endpoint cell is never deleted — Unity's carve-then-re-add resets per-cell state ("§2 edge case" in the plan).
   - **Carve before increment, per sample** — within one `addSample` all rays are carved, _then_ all endpoints are incremented. So deeper-carves-nearer holds **across** samples (a later sample's ray erases an earlier sample's endpoint) but **not within** one sample: an endpoint observed by one point survives another point's overlapping ray in that same sample, regardless of point order. A single interleaved pass would be order-dependent (the deeper ray would erase the nearer endpoint only if processed second).
4. **`getCellCenter` is round-consistent** (`cell · cellSizeM`) — deliberately NOT Unity's `CellToWorldPos` (+half cell), which is off by half a cell under round-quantization.
5. **Chunk index invariant** — `chunks` (Map chunkKey → Set of cellKeys, 16³ cells per chunk) is updated in the same operations that mutate `cells` (`increment` adds on first observation, `carve` removes + drops empty sets, `clear` resets), so the windowed queries can never serve a carved cell or miss an added one. Memory ≈ one Set entry per cell + one Map entry per chunk (~hundreds of chunks per 100k cells).
6. **Memory & keying** — unbounded `Map` keyed by a **numeric packed cell key** (was the string `"x,y,z"` — switched 2026-06-30 to drop the per-lookup string allocation in the `addSample` / carve hot paths). The packing uses 17-bit fields, so cell coordinates must be within **±65 535** (≈ ±9.8 km at 0.15 m — far beyond any real session); `addSample` skips any point whose cell falls outside that range. Same unboundedness as Unity's dictionary (whose far denser field tests never surfaced problems); carving recycles cells in revisited areas.
   - **Envelope guard on every key-touching path (2026-07-04, PR #144/#145/#147 reviews)** — beyond the envelope the base-2^17 packing ALIASES (e.g. `key([0, 65537, z]) === key([1, −65535, z])`), so an unguarded caller would read/delete an unrelated cell's record. All public lookups (`getCellPoint`, `getCellColor`, `raycast` endpoints) now answer `null` outside the envelope, and `addSample` skips **carving** when the camera cell itself is out of range (endpoints keep their own existing check). Pinned by the "key-range envelope hardening" tests.
   - **Note the occluder mesher is stricter:** `meshOccupiedCells` packs half-lattice corner keys (`2·coord ± 1`) and therefore skips cells beyond **±32 767** (≈ ±4.9 km) — cells the grid legally stores between the two bounds are silently unmeshed (documented in `occupancy-mesher.ts.md` §Invariants; unreachable in a real session).
7. **Replay throughput** — `addSample` is O(points × ray cells); replay re-dispatches faster than 1 Hz but runs on desktop.
8. **Flat per-cell records + carve dedupe (Step 3, 2026-07-03 fps plan)** — `CellRecord` is one plain object of eight scalars (no stored `cell` tuple, no nested sum arrays; was ~5 heap objects / ~201 B per cell), roughly halving the grid's long-lived heap and the old-gen GC mark workload on a long walk. Within one `addSample`, free-space carving runs once per **unique endpoint cell** instead of once per point — byte-identical grid state (carve-before-increment makes the repeat an exact no-op; the duplicate-scaling property test pins the contract) at ~2–5× less bresenham+Map work at gridSize 32. The per-point endpoint scratch-array reuse (plan Step 3.3) and a deeper SoA/dense-chunk storage rewrite (Step 3.4) were **deliberately deferred** pending profiles.

## Examples

```ts
const grid = new OccupancyGrid();
let last: DepthSample | null = null;
const store = storeRef.get();
store.subscribe(() => {
  const sample = selectLatestDepthSample(store.getState());
  if (sample && sample !== last) {
    last = sample;
    grid.addSample(sample);
  }
});
const cells = grid.getOccupiedCells(2); // noise-filtered
const hit = grid.raycast(cameraPos, forwardPoint); // cursor placement
```

## Tests

- `occupancy-grid.test.ts` — construction validation, add/skip paths (old recordings, invalid points, non-finite camera), counts, carve-stop protection, same-cell deviation, scene-change carving, intra-sample point-order independence, center formula, raycast hit/miss/min-observations, clear, and the Iter-8 cell-color block (verbatim single observation, rounded averaging, no dilution by color-less observations, non-finite rejection, clear drops colors).
- `occupancy-grid.property.test.ts` — fast-check: quantization↔center within `cellSizeM/2` per axis (guards against Unity's half-cell offset); observed cells survive any repeat observation count (incl. degenerate same-cell case); a nearer on-ray cell is carved iff it is at least `carveStopCells` in front of a deeper observation; cell color equals the rounded per-channel mean of exactly the colored observations (Iter 8); `getOccupiedCellsFlat` ≡ flattened tuple snapshot (Step 1.3); `getOccupiedCellsWithin` ≡ brute-force radius filter (Step 2); duplicate points scale counts without changing the occupied set (Step 3.2 carve-dedupe contract); `unpackCellKey` is the exact inverse of `cellKey` over the packable envelope (Step 3.1).
- `occupancy-grid.perf.test.ts` — long-session scaling harness (fps-plan acceptance): a fully-interior windowed query returns the SAME cell count at 20k and 200k total cells (deterministic gate) and stays within a loose 4× wall-clock ratio (an O(total-cells) regression would show ~10×); informational bytes-per-cell probe.
